Apuesto a que te has encontrado muchas veces pensando: **"He leído algo que me ayudará con este problema"**, pero no recuerdas dónde lo viste y no tienes forma de encontrarlo.

### GitHub al rescate
¿Qué? ¿GitHub? y la respuesta es un gran SÍ. Puedes crear un repositorio de GitHub y utilizar las ediciones para hacer un seguimiento de lo que lees, ves o escuchas en Internet.

Así que empecé a usar los issues de GitHub, que según la información oficial, son:

> Usar issues para rastrear ideas, mejoras, tareas o errores de trabajo en GitHub.


### ¿Cómo hacerlo?
1. [Crear](https://github.com/join) una cuenta de GitHub (pero apuesto a que ya tienes una). La mía es: [fmontes](https://github.com/fmontes)
2. 2. [Crear un repo](https://help.github.com/en/articles/create-a-repo), el mío lo he llamado [learning-tracking](https://github.com/fmontes/learning-tracking).
3. Para cada pieza de contenido, quieres guardar [crear un issue](https://help.github.com/en/articles/creating-an-issue).
4. Y eso es prácticamente todo.

<Image src="/static/images/blog/how-to-track-your-online-learning-withub/github-issues-list.jpg" alt="GitHub Issues List" width="653" height="310" />

### ¿Qué añadir a las cuestiones?
Cuando consumo algún contenido online, siempre quiero recordar ciertas partes, así que en mis issues, normalmente añado

1. Fragmentos de código con comentarios al respecto
2. Citas del artículo
3. Notas con conclusiones
4. Enlaces, por supuesto
5. Fuente

#### Los temas de GitHub son compatibles con Markdown

Markdown te permitirá añadir:

1. Fragmentos de código
2. Comentarios
3. Citas
4. Enlaces
5. Imágenes
6. Tablas
7. Y así [mucho más](https://guides.github.com/features/mastering-markdown/).

### Usar etiquetas
Puedes crear y asignar a las incidencias tantas etiquetas como quieras. Sería conveniente que usaras etiquetas en tus asuntos para poder filtrarlos después.

<Image src="/static/images/blog/how-to-track-your-online-learning-withub/github-issues-tags.jpg" alt="GitHub Issues Tags" width="699" height="354" />

### Búsqueda
No sólo podrás filtrar por etiquetas, GitHub tiene una función de búsqueda realmente potente que encontrará palabras clave en el título, las etiquetas e incluso el contenido de tus incidencias. ¡No puedes volver a perder nada!

### Conclusión
He probado cualquier aplicación de notas que se te ocurra, pero ninguna de ellas me funciona. Con este enfoque, he visto buenos resultados:

1. No he vuelto a perder contenido
2. Los fragmentos con notas son útiles
3. Al ser público (puedes hacerlo privado) permite que tus compañeros vean lo que estás haciendo
4. Mejora tu marca de desarrollador. La gente puede ver lo que estás haciendo 😉 .
